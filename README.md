A production-grade, conversational P2P exchange engine tailored for the Nigerian market. This bot facilitates direct USDT (TRC20) to Naira (NGN) trades using a human-centric approach, localized Nigerian Pidgin responses, and robust security protocols.

---

## 🌟 Professional Features

### 🇳🇬 Localized Experience
- **Authentic Communication**: Over 200+ response variations in **Nigerian Pidgin** and local slang.
- **Human-Centric**: Designed to feel like a real trader ("The Plug"), reducing the "robotic" friction of traditional bots.

### 🛡️ Security & Reliability
- **Double-Confirmation**: Critical financial data (wallet addresses, account numbers) requires double-entry verification.
- **Stateless Architecture**: No internal balance holding; reduces risk and overhead.
- **Permanent Records**: Integrated with **Supabase (PostgreSQL)** to ensure trade history persists forever.
- **Idempotent Fulfillment**: Admin approval logic prevents double-crediting or accidental notifications.

### ⚙️ Operational Efficiency
- **Smart Reminders**: Automated customer engagement messages sent every 90 days.
- **Natural Language Processing**: Recognizes intent from normal words (e.g., "How far", "Buy", "Rates") using optimized regex patterns.
- **Manual Fulfillment**: Streamlined admin dashboard with one-click approval buttons.

---

## 🏗️ System Architecture

The bot follows a **Stateless Event-Driven** pattern:

1. **User Interaction**: Triggered via `ConversationHandler` states.
2. **Data Layer**: All persistent data (Users, Transactions, Settings) is managed via the `supabase-py` client.
3. **Infrastructure**: Includes a background thread running a minimal HTTP server to satisfy **Render's** health-check requirements for free-tier hosting.

## 🚀 Deployment Guide (Expert)

### 1. Database Provisioning
Create a new project on **Supabase** and execute the following in the **SQL Editor**:

```sql
-- Create User Profiles
CREATE TABLE users (
    user_id BIGINT PRIMARY KEY,
    phone TEXT, 
    bank_name TEXT, 
    bank_account_name TEXT, 
    bank_account_number TEXT, 
    crypto_address TEXT,
    naira_balance BIGINT DEFAULT 0, 
    usdt_balance REAL DEFAULT 0,
    registered_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Create Transaction Ledger
CREATE TABLE transactions (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    user_id BIGINT REFERENCES users(user_id),
    type TEXT, 
    amount_currency TEXT, 
    amount REAL, 
    rate REAL,
    status TEXT, 
    tx_hash TEXT, 
    details TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Create System Settings
CREATE TABLE settings (
    key TEXT PRIMARY KEY,
    value TEXT
);
```

### 2. Environment Configuration
Configure the following variables in your hosting provider (Render/Koyeb) or local `.env`:

| Key | Description | Example |
|-----|-------------|---------|
| `BOT_TOKEN` | Telegram Bot API Key | `12345:ABC...` |
| `ADMIN_ID` | Telegram User ID for Owner | `987654321` |
| `SUPABASE_URL` | Supabase Project URL | `https://xyz.supabase.co` |
| `SUPABASE_KEY` | Supabase Service/Anon Key | `eyJhbGci...` |
| `TRONGRID_API_KEY` | API Key for Blockchain Verification | `f9a...` |
| `YOUR_BANK_NAME` | Displayed to buyers | `Access Bank` |
| `YOUR_USDT_WALLET` | Your receiving TRC20 address | `TQ...` |

### 3. Hosting (Render Free Tier)
- **Service Type**: Web Service
- **Build Command**: `pip install -r requirements.txt`
- **Start Command**: `python bot.py`

### 4. Security Provisioning (Recommended)
Paste this into the Supabase SQL Editor after creating tables to lock down your data:

```sql
ALTER TABLE users ENABLE ROW LEVEL SECURITY;
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;
ALTER TABLE settings ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Bot full access" ON users FOR ALL TO service_role USING (true) WITH CHECK (true);
CREATE POLICY "Bot full access" ON transactions FOR ALL TO service_role USING (true) WITH CHECK (true);
CREATE POLICY "Bot full access" ON settings FOR ALL TO service_role USING (true) WITH CHECK (true);
```

## 👨‍💼 Administration

- **Synchronize Rates**: `/setrate <buy_price> <sell_price>`
- **Fulfillment**: Click **Approved** on trade notifications once you verify funds in your actual bank/wallet app.
- **Engagement**: Use `/broadcast <message>` to reach all registered users.
